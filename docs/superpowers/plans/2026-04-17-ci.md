# CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a GitHub Actions CI workflow that runs frontend and backend checks on every push and PR to main.

**Architecture:** One workflow file with two parallel jobs — `frontend` (typecheck + lint + test + build) and `backend` (prisma generate + test + build). No deployment, no secrets required. Vercel auto-deploy already handles the frontend.

**Tech Stack:** GitHub Actions, `actions/checkout@v4`, `actions/setup-node@v4`, Node.js 22.x

## Task 1: Create CI Workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create the `.github/workflows/` directory structure and write `ci.yml`**

```bash
mkdir -p .github/workflows
```

Then create `.github/workflows/ci.yml` with this exact content:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  frontend:
    name: Frontend
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '22.x'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Type check
        run: npm run typecheck

      - name: Lint
        run: npm run lint

      - name: Test
        run: npm run test:run

      - name: Build
        run: npm run build

  backend:
    name: Backend
    runs-on: ubuntu-latest
    defaults:
      run:
        working-directory: server
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '22.x'
          cache: 'npm'
          cache-dependency-path: server/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Generate Prisma client
        run: npx prisma generate

      - name: Test
        run: npm run test:run

      - name: Build
        run: npm run build
```

- [ ] **Step 2: Validate YAML syntax locally**

```bash
python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml')); print('Valid YAML')"
```

Expected: `Valid YAML`. If `ModuleNotFoundError`, try `python3 -m pip install pyyaml` or skip — GitHub will catch syntax errors on push.

- [ ] **Step 3: Run all frontend checks locally to confirm they pass in this state**

```bash
npm run typecheck && npm run lint && npm run test:run && npm run build 2>&1 | tail -3
```

Expected: typecheck and lint exit 0 (lint shows warnings, no errors), test shows `6 passed`, build shows `✓ built in X.XXs`.

- [ ] **Step 4: Run all backend checks locally**

```bash
cd server && npm run test:run 2>&1 | tail -4
```

Expected: `67 passed`, no `❌` lines, exit 0. (`prisma generate` is already done locally so you can skip that step here.)

- [ ] **Step 5: Commit**

```bash
cd "$(git rev-parse --show-toplevel)"
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions workflow for frontend and backend checks"
```

- [ ] **Step 6: Push to GitHub and verify workflow triggers**

```bash
git push
```

Then open `https://github.com/LAI-wen/SmartCapital/actions` in a browser. You should see a new workflow run called "CI" for the push to main. Both `Frontend` and `Backend` jobs should appear and turn green within ~2–3 minutes.

If a job fails on GitHub but passes locally, common causes:
- **`npm ci` fails** — `package-lock.json` is out of sync. Fix: run `npm install` locally, commit the updated lock file.
- **`prisma generate` fails** — `prisma/schema.prisma` not found. Verify the path with `ls server/prisma/`.
- **Backend `tsc` fails** — Prisma generated types missing. The `Generate Prisma client` step should prevent this; if it still fails, check the prisma generate output for errors.
