# CI — Round 3 Design

**Date:** 2026-04-17
**Scope:** GitHub Actions CI workflow — frontend and backend checks on push/PR to main
**Out of scope:** Deployment automation (Vercel handles frontend, Fly.io backend is manual)

---

## Problem

`smartcapital` has no CI. Broken code can land on main without any automated gate. For a portfolio project, a green CI badge and a workflow file are table-stakes signals of engineering maturity.

---

## Workflow

**File:** `.github/workflows/ci.yml`

**Trigger:**
```yaml
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
```

Runs on every push to main and every PR targeting main.

---

## Jobs

Two parallel jobs — neither depends on the other.

### `frontend`

Runs on `ubuntu-latest`, Node.js `22.x`.

Steps:
1. `actions/checkout@v4`
2. `actions/setup-node@v4` with `node-version: '22.x'` and `cache: 'npm'`
3. `npm ci`
4. `npm run typecheck`
5. `npm run lint`
6. `npm run test:run`
7. `npm run build`

All steps must pass. If any fails, the job is red and merging is blocked.

### `backend`

Runs on `ubuntu-latest`, Node.js `22.x`.

Steps:
1. `actions/checkout@v4`
2. `actions/setup-node@v4` with `node-version: '22.x'` and `cache: 'npm'`, `cache-dependency-path: 'server/package-lock.json'`
3. `npm ci` (in `server/` via `working-directory: server`)
4. `npm run test:run` (in `server/`)
5. `npm run build` (in `server/`)

All steps must pass.

---

## Completion Criteria

- `.github/workflows/ci.yml` exists and is valid YAML
- A push to main triggers both jobs on GitHub Actions
- Both jobs pass on the current codebase
- A failing frontend test or lint error causes the frontend job to fail (verified locally by intentionally breaking a test and checking `npm run test:run` exits non-zero)
